# 九州征途（SLG）设计文档

类《率土之滨》的沙盘策略游戏。前端 Phaser 3 渲染大世界 + Vue/Quasar 做 UI 层，后端 Go。
分阶段实现，**阶段二（玩法深化）已完成**：NPC 城池攻坚、建筑体系、体力系统、武将招募
均已落地（详见「四、阶段二详细规划」）。下一步进入阶段三（Go 后端接入）。

## 二、MVP 架构（阶段一）

严格分层（仿 ironfist 的逻辑/渲染解耦）。游戏整体（世界 + 全部 UI）由 Phaser 实现，
Vue 仅充当路由挂载壳：

```text
frontend/src/games/slg/
├── SlgPage.vue                 # 纯挂载壳：创建/销毁 Phaser 实例 + 路由返回
└── game/
    ├── GameConstants.js        # 全部数值常量
    ├── SlgGame.js              # Phaser 实例工厂（createSlgGame，启动双场景）
    ├── core/                   # 纯逻辑层，零 Phaser 依赖
    │   ├── MapGenerator.js     # 种子化地图生成（mulberry32，同 seed 同图；含守军兵种）
    │   ├── pathfind.js         # 网格寻路（BFS 四方向，绕障）
    │   ├── battle.js           # 确定性战斗结算（无随机；兵种克制倍率由调用方叠入攻方武力）
    │   └── GameState.js        # 资源/行军/占领/存档 + 事件发射
    └── scenes/
        ├── WorldScene.js       # 大世界渲染（运行时生成贴图、相机、行军动画）
        └── UIScene.js          # HUD 场景：资源栏/行军列表/地块面板/弹窗/提示
```

- **数据流**：`GameState`（唯一事实源）→ 事件 → WorldScene / UIScene 重绘；
  用户操作 → UIScene 调 `state.march()/recruit()/...` → 事件回流。
- **输入仲裁**：UIScene 暴露 `hitTest(x,y)`，WorldScene 处理指针前先询问，
  避免点击 HUD 时穿透到地图（平移/选地块）。
- **确定性**：地图由 seed 决定、战斗无随机 —— 存档只记 seed + 增量，
  也为阶段三服务端校验/回放打基础（同 ironfist 的 replay 思路）。

## 三、核心规则（MVP 数值）

- **地图**：48×48 方格。类型：平原/农田(粮)、森林(木)、丘陵(石)、山地(铁)、湖泊(不可通行)、NPC 城池。
  等级按到地图中心距离分带：外圈 Lv.1 → 中心 Lv.5（±1 抖动）。
- **时间加速**：`TIME_SCALE = 60`，1 真实秒 = 60 游戏秒（游戏内 1 小时 ≈ 真实 1 分钟）。
- **产量**：地块 `level × 100/小时` 对应资源；所有领地另产铜币 `level × 20/小时`。
  NPC 城池四种资源各 `level × 50/小时`。
- **守军**：`level×(level+1)/2 × 100`（Lv.1=100 … Lv.5=1500）；NPC 城池固定 3000。
- **战斗**：`攻方战力 = 兵力 × (1 + 武力/100)`，`守方战力 = 守军 × (1 + 守防/100)`。
  胜：攻方损失 `兵力 × (守/攻) × 0.5`，守军全灭；败：攻方损失 70%。战败也可磨血（守军 -劣势比×50%）。
- **占领规则**：目标必须与己方领地八向相邻；领地数上限 `8 + 主城等级 × 4`。
- **行军**：沿网格逐格移动（BFS **四方向**寻路——只走上下左右、不走对角线，绕开湖泊等
  不可通行地块），从主城出发、战后原路折返。每格耗时 =
  `MARCH_TILE_SECONDS_AT_REF × (100 / 有效速度)` 游戏秒；有效速度 100 时每格 ≈ 真实 1 秒。
  **有效速度 = 武将速度 + 兵种加成（骑兵 +30）**；合击按全队最慢者计时。
  详见 [core/pathfind.js](../frontend/src/games/slg/game/core/pathfind.js) 与 `tileMarchSeconds`。
- **兵种与克制**（率土式三角克制）：**盾克弓、弓克枪、枪克盾**（骑兵不参与克制，
  只享行军速度加成）。武将出生即固定兵种（`STARTER_GENERALS`/`RECRUITABLE_GENERALS`
  的 `troopType` 字段）。克制方战斗武力 ×1.25，被克方 ×0.85（`counterMult`）。
  地块守军也由种子确定性分配兵种（`garrisonType`，独立 rng 生成，不扰动地形/城池布局），
  出征前地块面板与出征弹窗都会提示"克制/被克"。详见 `TROOP_TYPES`。
- **武将**：出生固定兵种；带兵上限 `1000 + (lv-1)×200`；经验升级每级 +2 武 +2 防。
  征兵 2 粮/兵（需在城待命）。
- **主城**：Lv.1~5，升级花费 500×2^(lv-2) 的铜/木/石，提升领地上限。
- **城池攻坚**（阶段二已实装）：
  - **合击**：出征可多选武将，全队按最慢速度行军，合兵结算（总兵力相加、武力按兵力加权平均），
    伤亡与经验按出兵比例分摊。
  - **挫伤与回复**：战败也会消耗守军；未占领地块的守军按每游戏小时「上限的 10%」缓慢回复，
    形成波次攻坚节奏。
  - **掠夺**：攻克 NPC 城池一次性获得 铜2000/粮5000/木铁石各3000。
  - **一统**：八座 NPC 城池尽克触发「天下一统」（单机版胜利目标，可继续经营）。
- **存档**：localStorage（key `slg:save:v1`，数据版本已升至 **v4**：含建筑、武将体力与免费招募次数），
  每 10 秒 + 离开页面时自动保存；离线按真实时长 × TIME_SCALE 折算补产出，封顶 8 游戏小时。
  旧 v1~v3 存档自动迁移（建筑默认 1 级、体力补满、免费招募次数记为 0——旧存档已获得过起手三武将）。

## 四、阶段二详细规划（已完成）

### 4.1 建筑体系 ✅（已实装）

与地块产出（率土的「土地」）不同，建筑是**主城内部**的四条独立强化线，
效果覆盖全局，是「回城之后做什么」的核心答案。
入口：点主城地块 →「🏛️ 建筑管理」。四条线默认各 1 级（1 级即带基础加成）。

| 建筑 | 效果 | 主资源 |
| --- | --- | --- |
| 🌾 粮仓 Granary | 全资源产出 ×(1 + 0.05×等级)（含铜币，与地块产量叠乘） | 粮 + 木 |
| ⚔️ 兵营 Barracks | 所有武将带兵上限 `troopCapOf(lv) + 100×等级` | 木 + 铁 |
| 🎯 校场 Training | 在城（`idle`）武将被动获得经验 `20×等级/游戏小时`（挂机练级，无需一直出征） | 石 + 铜 |
| 🔨 铁匠坊 Forge | 所有武将战斗武力/防御 `+1×等级`（结算时叠加，独立于自身升级加成） | 铁 + 石 |

- **等级**：1~5，且 `建筑等级 ≤ 主城等级`（主城仍是节奏阀门，逼玩家先升主城再堆建筑）。
- **升级花费**：`300 × 2^(toLv-2)` 的两种主资源（同主城升级公式的思路，成本更低但要建四条线）。
- **架构**：`GameState.buildings = { granary, barracks, training, forge }`（默认各 1 级）；
  新增 `upgradeBuilding(type)`；`yieldPerHour()` 乘粮仓系数；带兵上限改用
  `state.troopCap(g)`（原常量 `troopCapOf` 保留作为兵营 0 级基线，不再被 UI 直接调用）；
  校场加成在 `_produce(gameSeconds)` 内对 `idleGenerals()` 调 `_gainExp`；
  铁匠坊加成在 `_arriveAndFight` 计算 `avgAtk`/`stats.def` 时叠加。
- **UI**：主城地块面板新增「🏛️ 建筑」按钮 → 弹窗四行（同武将面板结构：等级/当前效果/下一级花费/升级按钮）。

### 4.2 体力系统 ✅（已实装）

限制出征频率的核心节奏器，武将级资源（不是全局共享）：

- `staminaMax = 100`；出征固定消耗 `MARCH_STAMINA_COST = 20`（不按距离，MVP 简化，够连打 5 次）。
- 回复：`STAMINA_REGEN_PER_HOUR = 5`（游戏小时），回满需 20 游戏小时 ≈ 20 真实分钟
  （`TIME_SCALE=60` 下 1 游戏小时 ≈ 1 真实分钟），无论武将在城还是行军中都持续回复。
- `march()` 校验：参战武将中任意一人体力 `< COST` 则整体拒绝出征，
  提示「体力不足，还需等待」，避免无脑连点出征把攻坚变成体力盲打。
- **架构**：`generals[].stamina` 新字段；出征时批量扣除；回复逻辑挂在 `_produce(gameSeconds)`
  （与守军回复 `_regenGarrisons` 同一节拍）。
- **UI**：武将面板/出征弹窗每行加一条体力条；体力不足的武将出征弹窗里标灰并注明剩余回复时间。

### 4.3 武将招募 ✅（已实装）

铜币的主要消耗口（此前只进不出）。入口：武将面板右上「🎲 招募」。

- **花费**：每次 `RECRUIT_COST_COIN = 800` 铜币；开局赠送 `FREE_RECRUIT_COUNT = 1` 次免费招募
  （不再固定起手 3 武将，改为送一次抽卡机会，UI 首次进入自动弹出招募面板引导抽第一名武将；
  该次抽到「新武将」时额外获得起始兵力 `INITIAL_TROOPS`，付费招募的新武将仍为 0 兵需另行征兵）。
- **品质与概率**：普通 50% / 精良 30% / 精锐 15% / 王牌 5%（`GENERAL_QUALITY`）。
  先 roll 品质，再从该档的 `RECRUITABLE_GENERALS` 池随机取一名。
- **阵容上限** `MAX_GENERALS = 8`。
- **重复/满员 → 觉醒**：抽到已拥有的武将（或阵容已满抽到新武将）转为觉醒，
  给目标武将 +3 武 +3 防（`AWAKEN_ATK/DEF`），记录觉醒次数（面板显示 `✦n`），避免纯重复浪费。
- **随机性**：单机版用 `Math.random`（无需确定性；阶段三联网后由服务端掷点）。
- **存档**：招募武将不在初始阵容，`GameState.load()` 按 `findGeneralTemplate(id)` 重建后
  再套用存档动态字段（等级/兵力/体力/觉醒）；`awaken` 字段随存档保存（仍在 v3 内，向后兼容）。
- **UI**：武将列表改为可滚动（`_makeScrollRegion` 通用滚动组件，与战报共用），
  名字按品质染色。

### 4.4 存档版本 ✅（已升至 v4）

存档已升级到 **v4**：新增 `freeRecruits` 字段（开局改为送一次免费招募而非固定 3 武将，
v1~v3 迁移时记为 0）；此前 v3 新增 `buildings` 字段（v1/v2 迁移时默认各级为 1）、
`generals[].stamina` 字段（迁移时因 `Object.assign` 不覆盖缺失键而保留构造时的 `STAMINA_MAX`）。
沿用 `GameState.load()` 按版本号迁移的模式（v1→v2 的 `marches[].generalId → generalIds[]` 迁移仍保留）。

## 五、已知取舍（MVP 有意简化）

- 行军已改为网格寻路（四方向、绕障、逐格计时）。~~直线距离计费~~ ✅
- 兵种目前只有「武将固定兵种」一种分配方式（~~玩家自由切换~~ / ~~混编部队~~ 已评估搁置，
  见对话记录：切换/混编会显著增加 UI 与结算复杂度，MVP 阶段收益不足）。
- 所有部队从主城出发（率土规则），暂无「营帐/前哨」前进基地。
- 免战/土地掉级机制推迟到阶段四（多人同盘）：单机版只有静态 NPC 守军，
  不存在「被其他玩家反攻」的场景，免战在纯单机语境下收益很低。
- 单行军队列限制为「每武将一队」，三武将即最多三路并进。
